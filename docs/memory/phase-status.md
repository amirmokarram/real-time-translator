**DONE (all 2026-06-04):**
- **Phase 1 ‚Äî Scaffold:** Angular 21 + Electron 42 wired; IPC bridge (contextBridge/preload, typed `ElectronAPI`); 6 translation providers; settings persisted to `userData/settings.json`; dark dual-pane UI.
- **Phase 2 ‚Äî System audio capture:** `AudioService` does `getUserMedia` desktop-loopback in renderer; `AudioContext`+`AnalyserNode` RMS level meter. (See gotchas: video-track discard.)
  - **Source picker (2026-06-04):** the audio-bar ComboBox now offers **System Audio** + **each microphone** (not per-screen rows). `AudioSource` has a `kind` ('system'|'microphone'); ids are prefixed `system:<screenId>` / `mic:<deviceId>`. `getSources()` (main) returns one System Audio entry; mics enumerated in renderer via `enumerateDevices` (with a getUserMedia probe so labels populate); `acquireStream()` branches on `kind`. NOTE: `settings.audio.selectedSourceId` is still never written/read ‚Äî selection does not persist across restarts (pre-existing gap, offered as follow-up).
- **Phase 3 ‚Äî STT (DeepGram):** `TranscriptionService` streams via DeepGram WebSocket; `MediaRecorder` WebM/Opus chunks; auto-reconnect. Settings has DeepGram key + Test Connection. (Free tier 12k min/yr, key at console.deepgram.com.)
  - **Sentence segmentation (2026-06-06):** DeepGram segments on *silence*, not grammar, so naive per-`is_final` commits fragmented sentences across rows. Now: buffer `is_final` fragments in `pendingFinal`, peel off complete sentences (`.!?`) via `drainSentences()` the moment the next begins ‚Üí one row = one grammatical sentence, committed in real time while listening continues. Trailing partial joins the next utterance or commits via a 4s idle timer / on stop. `endpointing=800` (brief clause pauses don't split); `utterance_end_ms=1000` UtteranceEnd backstop. **Coalescing gotcha:** committed sentences go through a `pendingSentences` queue + `finalVersion` counter (NOT a single value signal) ‚Äî a burst committed in one tick would otherwise be lost to Angular signal coalescing. Translator drains the queue **sequentially** (`drainSttQueue`) ‚Üí one row each, in order, no overlapping `translate()` calls. `lastFinalText` now exists only as a live-panel display fallback.
- **Phase 3.5 ‚Äî Meeting UX:** chat-style history (newest at bottom, auto-scroll, synced EN|FA rows); live panel replaces textarea while capturing; tabbed settings (Translation / Speech / Display).
- **LibreTranslate fix:** Content-Length + error-field handling (see gotchas).
- **Phase 4a ‚Äî Overlay mode:** floating always-on-top transparent subtitle window at `#/overlay`; translation events BROADCAST to all windows from main; click-through with hover-to-wake toolbar; header toggle button.
- **Phase 4b ‚Äî History export:** `ExportService` ‚Üí TXT transcript or SRT subtitles via native save dialog (`export:save` IPC); export popover in audio bar.
- **Phase 5 ‚Äî Assist mode (2026-06-07, branch `feature/assist-mode`):** LLM Q&A *about the conversation*, kept separate from translation. **Row selection layer** in translator (click/shift-click ‚Üí highlight; per-row hover copy retained). Selection bar is **Ask-only** ‚Üí opens a **slide-in panel** (lives in app shell `app.ts`, so it works on any route, not just translator; excluded from overlay window). Context sent to the LLM is **English-only** (source of truth). Panel: streaming chat, quick actions (Summarize/Explain/Key terms) when context present, collapsible context chip, multi-turn follow-ups, new-thread/close. **Header "Assist" button** opens it context-free for free-form chat. Backend mirrors translation: `electron/assist/` ‚Äî `IAssistProvider` + `composeAssistPrompt` (in `electron/prompts.ts`), registry, `assist:ask` IPC streaming `assist:chunk`/`assist:complete` **to the calling window only** (not broadcast). **4 providers:** Claude + OpenAI (reuse the matching translation provider's API key ‚Äî nothing to re-enter), **Ollama** (local `/api/chat`, newline-delimited JSON), **OpenAI-compatible** (generic `/chat/completions` for **Docker Model Runner**, LM Studio, vLLM, llama.cpp, LocalAI). New `assist: { provider, model, endpoint }` settings section (endpoint only for local; defaults seeded per-provider on switch). **Settings ‚Üí Assist** tab: provider/model selects, local providers get free-text model + endpoint + setup how-to, **Test Connection** (`assist:validate` does a minimal ping), API-key-missing warning for cloud. `AssistService` (renderer) holds thread/streaming/context signals.
  - **Gotcha (fixed):** native `<select>` with `[value]` does NOT reliably reflect the selected option when options render as the DOM is (re)created (tab switch / app reload) ‚Üí dropdown visually resets to first item though the saved value is correct. Fix: use `[ngModel]` + `[ngModelOptions]="{standalone:true}"` (no wrapping `<form>`). Applies to all settings selects.
  - **Prompt customization:** assist + translation system prompts editable in Settings (Assist / Translation tabs) with Reset-to-default; stored in `prompts:{assist,translation}` (empty ‚Üí built-in default). `electron/prompts.ts` is the single source: `DEFAULT_*` consts, `composeAssistPrompt(base,context)`, `resolveTranslationPrompt(custom)` ‚Äî **replaced the old `buildSystemPrompt`**. `AssistRequest`/`TranslationRequest` carry `systemPrompt`; all 4 assist providers + Claude/OpenAI translation read it (translation prompt only affects LLM providers ‚Äî MT ones ignore it). `prompts:get-defaults` IPC backs editor prefill/reset.
  - **Default assist prompt = interview assistant:** English-only (no Persian unless asked), A2‚ÄìB1, human/conversational. With transcript context ‚Üí strict **two-section scannable Markdown**: `### üîµ What they mean` + `### üü¢ Say this` (the read-aloud answer). Quick-action buttons retuned to **Explain / Answer / Key terms**.
  - **Free-chat vs interview:** `composeAssistPrompt` is context-aware ‚Äî header Assist (no context) appends a "free chat, drop the two-section format, answer directly" footer; selection Ask appends the transcript so the interview format applies. Single customizable prompt; behaviour varies by context presence.
  - **Markdown rendering:** assistant replies + live stream go through `MarkdownPipe` (`marked` + `DOMPurify` sanitize ‚Üí `SafeHtml`); user messages stay plain. Styled via `:host ::ng-deep .markdown` (innerHTML nodes carry no encapsulation attrs). **All links open in the system browser:** DOMPurify hook stamps `target=_blank`/`rel`, and main's `openLinksExternally` (`setWindowOpenHandler` + `will-navigate` ‚Üí `shell.openExternal`, deny in-place nav). Deps added: `marked`, `dompurify`.
  - **Punctuation normalization:** translation IPC handler `collapseTerminalPunctuation` collapses duplicated terminal marks (ÿüÿü/??/!! ‚Üí one, prefers Persian ÿü) ‚Äî fixes LibreTranslate/Argos NMT doubling the final `?` on questions.

**DONE ‚Äî Latency tuning, Phase A (2026-06-25):** user-facing knobs to make translations appear sooner (the pipeline was structurally ~one phrase behind the speaker). Four new `stt` settings (defaults = the old hardcoded values, so behaviour is unchanged until tuned): `endpointingMs` (DeepGram silence-before-final, was 800), `utteranceEndMs` (DeepGram UtteranceEnd backstop, was 1000, clamped ‚â•1000 by the API), `sentenceMaxWaitMs` (idle-commit fallback for un-punctuated tails, was the `SENTENCE_MAX_WAIT_MS=4000` static), `commitOnClause` (also split rows on `, ; :` not just `.!?`). Wiring: `endpointingMs`/`utteranceEndMs` flow through `SttStartOptions` ‚Üí DeepGram URL params; `sentenceMaxWaitMs`/`commitOnClause` read in `TranscriptionService.start()` ‚Üí drive the idle timer + segmentation regexes (built dynamically via `buildSentenceRe`/`buildEndsRe`). Schema added in BOTH `electron/settings-store.ts` (defaults+merge) and renderer `app.models.ts` (+ the `mockSettings` fallback in `electron-bridge.service.ts` ‚Äî easy to miss, breaks `tsc`). UI: a **"Latency tuning"** subsection in Settings ‚Üí Speech Recognition (number fields + clause toggle; DeepGram-only knobs hidden when Whisper selected). Whisper's pacing is server-side VAD + model size, so only `sentenceMaxWaitMs`/`commitOnClause` affect it. **Phase B (NOT built):** live partial translation ‚Äî translate in-progress interim text on a debounce, revising the row until the sentence finalizes (true follow-along; tuning alone can't remove the one-phrase lag). **Dev-run gotcha:** launching Electron via the agent's background-command harness gets the child processes reaped when the launcher returns (and `npm run electron:dev`'s `concurrently -k` then SIGTERMs `ng serve`). Run `ng serve` and Electron as **separate** background tasks, and launch Electron **detached** (`Start-Process node_modules\electron\dist\electron.exe . -WorkingDirectory <repo>` with `$env:ELECTRON_DEV='true'`) so it survives outside the harness process tree.
